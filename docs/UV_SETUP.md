# UV Setup Guide

This project uses **[uv](https://github.com/astral-sh/uv)**, a fast Python package manager and installer written in Rust.

## Why UV?

- ⚡ **Fast**: 10-100x faster than pip
- 🔒 **Deterministic**: Lock file ensures reproducible builds
- 🐍 **Complete**: Replaces pip, venv, virtualenv, and piptools
- 🖥️ **Zero config**: Works out of the box
- 🐳 **Container-friendly**: Minimal image size

## Installation

### On Your Machine

```bash
# macOS / Linux
curl -LsSf https://astral.sh/uv/install.sh | sh

# Windows
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"

# Add to PATH
export PATH="/root/.local/bin:$PATH"  # or equivalent for your shell
```

### In Docker

Already installed in `backend/Dockerfile`.

## Backend Development Workflow

### Install Dependencies

```bash
cd backend

# Create venv and install all dependencies (including dev)
uv sync

# Or install only production dependencies
uv sync --no-dev
```

### Run Commands

```bash
# Run the backend
uv run uvicorn app.main:app --reload

# Run tests
uv run pytest

# Run migrations
uv run alembic upgrade head
uv run alembic revision --autogenerate -m "description"

# Run linting/formatting
uv run ruff check . --fix
uv run black .
uv run mypy app/
```

### Add Dependencies

When adding new packages:

```bash
# Add to production dependencies
uv add fastapi-limiter

# Add to dev dependencies
uv add --dev pytest-cov

# Lock file automatically updates
```

The lock file (`uv.lock`) will be generated automatically.

## Lock File

**`backend/uv.lock`** — Deterministic dependency tree

- **Commit to git** — Ensures everyone uses the same versions
- **Auto-generated** — Don't edit manually
- **Reproducible** — `uv sync` always gives same environment

### Update Locked Dependencies

```bash
# Update all dependencies (respecting version constraints in pyproject.toml)
uv lock

# Then install
uv sync
```

## Docker Build

```bash
# Build with uv (faster, reproducible)
docker-compose build backend

# Run
docker-compose up
```

The Dockerfile:
1. Installs uv
2. Copies `pyproject.toml` and `uv.lock`
3. Runs `uv sync --frozen` (uses lock file exactly)
4. Runs the app

## Project Structure

**`backend/pyproject.toml`** — Package metadata & dependencies
```toml
[project]
dependencies = [
    "fastapi==0.115.2",
    "sqlalchemy==2.0.37",
    # ...
]

[project.optional-dependencies]
dev = [
    "pytest==8.3.3",
    # ...
]
```

**`backend/uv.lock`** — Locked versions of all transitive dependencies (generated by uv)

## Common Tasks

### Create Virtual Environment Without Installing

```bash
uv venv  # Creates .venv in current directory
```

### Check Outdated Packages

```bash
uv pip list --outdated
```

### See Dependency Tree

```bash
uv pip tree
```

### Export to requirements.txt (Legacy)

```bash
uv pip export --output-file requirements.txt
```

## Troubleshooting

### "uv: command not found"

Ensure uv is installed and in PATH:
```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
export PATH="/root/.local/bin:$PATH"
```

### "Failed to resolve dependencies"

Check `pyproject.toml` for version conflicts:
```bash
uv lock  # Will show the conflict
```

### Docker build fails with uv

Ensure `uv.lock` is committed:
```bash
git add backend/uv.lock
git commit -m "Add uv.lock"
```

## Migration from pip

For reference, here's what changed:

| Task | pip | uv |
|------|-----|-----|
| Create venv | `python -m venv venv` | `uv venv` |
| Install deps | `pip install -r requirements.txt` | `uv sync` |
| Add package | `pip install fastapi` | `uv add fastapi` |
| Run script | `python script.py` | `uv run script.py` |
| Lock versions | `pip-tools` | `uv lock` |
